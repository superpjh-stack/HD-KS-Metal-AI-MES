# 핵심 기능 명세 & 우선순위 (Core Feature Spec)

> 작성자: PM-Core (기획자 1) | 작성일: 2026-05-11 | 문서 버전: v1.0
> 분류: PDCA - Plan Phase - Feature Specification
> 영역: 입고관리 / 프레스 공정관리 / 일반 공정관리 / 출하관리 / 기준정보 / 사용자관리

---

## 1. 문서 목적

본 문서는 광성정밀 AI-MES 시스템의 **비-AI 핵심 업무 모듈**의 기능 명세, 사용자 스토리, 우선순위(MoSCoW)를 정의한다.
AI/예지보전/대시보드/KPI는 별도 문서(02-pm-ai-agent-spec.md, 03-pm-data-kpi-spec.md)를 참조한다.

---

## 2. 기능 우선순위 매트릭스 (MoSCoW)

| 분류 | 의미 | Phase |
|------|------|-------|
| **Must Have (M)** | 1차 사업 종료(2027-05) 시 반드시 동작 | Phase 1~3 |
| **Should Have (S)** | 가능한 한 1차에 포함, 2027 Q2까지 허용 | Phase 3~4 |
| **Could Have (C)** | 여건 되면 포함, 2차 사업 후보 | Phase 4 또는 이후 |
| **Won't Have (W)** | 본 사업 범위 외 | - |

## 3. 모듈별 기능 명세

### 3.1 원자재 입고관리 (Material Inbound Management)

#### 3.1.1 기능 목록

| 기능 ID | 기능명 | 우선순위 | 설명 |
|---------|--------|---------|------|
| INB-01 | 입고 등록 (QR/바코드) | M | 자재 입고 시 LOT 번호, 공급처, 수량 등록 |
| INB-02 | 입고 검사 결과 기록 | M | 외관/치수/재질 검사 결과 입력, 사진 첨부 |
| INB-03 | LOT 이력 조회 | M | LOT 단위로 입고→공정→출하 전체 이력 추적 |
| INB-04 | 공급처 품질 분석 | S | 공급처별 불량률, PPM 트렌드 자동 집계 |
| INB-05 | 입고 데이터 다운로드 | M | Excel/CSV 내보내기 (감사 대응) |
| INB-06 | 입고 검사 표준 관리 | M | 자재별 검사 항목, 합격 기준 관리 |
| INB-07 | 자재 부적합 격리 | M | 부적합 자재 자동 격리, 처리 상태 추적 |
| INB-08 | 코일 잔량 관리 | S | 프레스 코일 잔량 추적, 잔재 활용 |

#### 3.1.2 사용자 스토리

**Story INB-US-01**: 입고 검사원
> "원자재가 입고되면 트럭 옆에서 태블릿으로 QR을 스캔하고, 5분 안에 검사 결과까지 입력하고 싶다.
> 손이 더러우면 음성으로 '폭 30.0, 두께 1.2, 합격'이라고 말할 수 있어야 한다."

**Story INB-US-02**: 품질팀장
> "POSCO에서 받은 SPCC 코일이 3개월 동안 어떤 LOT으로 입고됐고, 어느 부품으로 나갔는지 한 화면에서 보고 싶다."

**Story INB-US-03**: 구매팀장
> "올해 공급처별 불량률 순위와 입고 지연 횟수를 월말 결산 회의 전에 자동 메일로 받고 싶다."

#### 3.1.3 수용 기준 (Acceptance Criteria)
- QR 스캔 후 입고 등록 완료까지 평균 60초 이내
- 한 LOT의 전체 이력 조회 응답시간 3초 이내
- 검사 합격/불합격 결정 시 다음 공정에 자동 알림
- 모든 입고 기록은 변경 이력 보존 (감사 추적성)

### 3.2 프레스 공정관리 (Press Process Management) — 핵심 모듈

> 프레스는 광성정밀의 **주력 공정**이며, IoT/AI 적용이 가장 중요한 영역이다.

#### 3.2.1 기능 목록

| 기능 ID | 기능명 | 우선순위 | 설명 |
|---------|--------|---------|------|
| PRS-01 | 작업 지시(WO) 표출 | M | 다음 작업 지시를 PRESS 옆 디스플레이에 자동 표출 |
| PRS-02 | 작업 시작/종료 기록 | M | 작업자 ID, 금형 ID, 코일 LOT, 시작/종료 시각 |
| PRS-03 | 실시간 SPM/스트로크 카운트 | M | 분당 타수, 누적 타수 자동 카운트 |
| PRS-04 | 작업조건 관리 | M | 부품×금형별 표준 조건(타발력, 슬라이드 위치, 다이쿠션) |
| PRS-05 | 공정 데이터 모니터링 | M | 진동/온도/전력/하중 실시간 차트 (1초 단위) |
| PRS-06 | 공정 이력 조회 | M | 시간별/LOT별 모든 측정값 재현 가능 |
| PRS-07 | 4M 변경 관리 | M | Man/Machine/Material/Method 변경 시 승인 워크플로 |
| PRS-08 | 금형 수명 관리 | M | 금형별 누적 타수, 점검 주기, 교체 예고 |
| PRS-09 | 자주검사 (Self-Inspection) | M | 작업자가 시간당 1회 측정값 입력, 한계 초과 시 경보 |
| PRS-10 | 불량 발생 등록 | M | 불량 유형, 수량, 사진, 추정 원인 입력 |
| PRS-11 | 정지 사유 관리 | M | 비가동 시간 사유 분류 (고장/금형교체/식사/회의 등) |
| PRS-12 | 프레스 예지보전 AI | M | 별도 문서(02-pm-ai-agent-spec.md) 참조 |

#### 3.2.2 사용자 스토리

**Story PRS-US-01**: 프레스 오퍼레이터 (현장 작업자)
> "교대 시작 시 내 사번을 찍으면, 오늘 내가 운전할 PRESS-03의 작업 지시와 표준 조건이 자동으로 뜨면 좋겠다.
> 작업 중에 측정값 1개만 입력하면 되고, 한계 벗어나면 빨간불이 켜져야 한다."

**Story PRS-US-02**: 작업반장
> "지금 가동 중인 8대 프레스 중 어떤 게 정상이고 어떤 게 이상한지 한 화면에서 보고 싶다.
> SPM이 평소보다 떨어지거나 진동이 튀면 노란/빨간 카드로 표시되어야 한다."

**Story PRS-US-03**: 금형 담당자
> "MOLD-A057의 현재 누적 타수가 47만 타이고, 점검 주기가 50만 타라면 3일 전부터 알림을 받고 싶다."

**Story PRS-US-04**: 품질팀
> "지난주 화요일 14시~15시 사이에 PRESS-05에서 나온 부품 LOT 번호와 그때의 진동/하중 그래프를 같이 보고 싶다."

#### 3.2.3 수용 기준
- 작업 시작 등록 후 5초 이내 표준 조건이 화면에 표출
- 측정값 한계 초과 시 1초 이내 경보음 + 화면 빨간색
- 1초 단위 IoT 데이터를 90일간 원본 유지, 그 이후 분당 평균으로 압축 1년 유지
- 4M 변경 이력은 영구 보존, 변경 전후 비교 화면 제공

### 3.3 일반 공정관리 (General Process Management)

> 프레스 이외의 후공정(탭핑, 디버링, 도금외주, 조립 등)을 다룬다.

#### 3.3.1 기능 목록

| 기능 ID | 기능명 | 우선순위 | 설명 |
|---------|--------|---------|------|
| PRC-01 | 공정실적 입력 | M | 공정별 생산수량, 양품/불량 수량 |
| PRC-02 | 작업조건 관리 | M | 공정별 표준 조건 (탭 깊이, 도금 두께 등) |
| PRC-03 | 표준 체크리스트 | M | 시업/종업 점검 체크리스트 디지털화 |
| PRC-04 | 공정 이력 조회 | M | LOT 단위 모든 공정 통과 이력 |
| PRC-05 | 공정 데이터 분석 | S | 공정별 불량 추이, 파라미터 영향도 |
| PRC-06 | 외주 공정 추적 | S | 도금/열처리 등 외주 출고/입고 LOT 추적 |
| PRC-07 | 작업자 자격관리 | S | 공정×작업자 자격(스킬맵), 무자격 작업 차단 |

#### 3.3.2 사용자 스토리

**Story PRC-US-01**: 후공정 작업자
> "탭핑 공정 완료 후 1분 안에 양품 950개, 불량 5개 입력하고 다음 LOT으로 넘어가고 싶다."

**Story PRC-US-02**: 품질 검사원
> "최종 출하검사 시점에 이 LOT이 거쳐온 모든 공정의 작업자, 작업 조건, 자주검사 결과를 1페이지에 보고 싶다."

### 3.4 출하관리 (Shipment Management)

#### 3.4.1 기능 목록

| 기능 ID | 기능명 | 우선순위 | 설명 |
|---------|--------|---------|------|
| SHP-01 | 출하 지시 관리 | M | 모회사 발주 정보, 납기, 수량, 차량 배정 |
| SHP-02 | 출하 검사 | M | 출하 전 최종 검사, 합격 시 출하 가능 |
| SHP-03 | 출하 등록 (QR) | M | 박스 단위 QR 스캔으로 출하 처리 |
| SHP-04 | 출하 이력 조회 | M | LOT/품번/일자별 출하 이력 |
| SHP-05 | 납기 현황 분석 | M | 납기 임박/지연 건 자동 알림 |
| SHP-06 | 출하 라벨 발행 | M | 모회사별 라벨 규격(VAN/EDI 호환) |
| SHP-07 | 회수 LOT 추적 | M | 모회사 클레임 발생 시 동일 LOT 출하 이력 즉시 조회 |
| SHP-08 | 출하 AI Agent | S | 별도 문서 참조 |

#### 3.4.2 사용자 스토리

**Story SHP-US-01**: 출하 담당자
> "모회사 차량이 도착하면 박스 QR 30개를 연속 스캔하고, 출하 라벨이 자동 출력되어 차량에 부착하면 좋겠다."

**Story SHP-US-02**: 영업/고객지원
> "현대 모비스에서 어제 보낸 LOT-20260510-A045 부품에 클레임이 들어왔다. 이 LOT으로 출하된 박스, 원자재, 작업자, 그때의 프레스 데이터를 30초 안에 모회사에게 답변하고 싶다."

#### 3.4.3 수용 기준
- 박스 QR 스캔 → 출하 처리 < 3초
- LOT 역추적 (출하→원자재) 30초 이내 완성된 보고서 제공
- 납기 24시간 전 미출하 건 자동 알림 (메일+카카오워크)

## 4. 통합 우선순위 요약

### 4.1 Phase별 기능 매핑

**Phase 1 (Foundation, 2026.07-08)**: MST-01~09, USR-01~05
**Phase 2 (Core MES, 2026.09-11)**: INB-01~03, PRS-01~11, PRC-01~04, SHP-01~07
**Phase 3 (AI Layer, 2026.12~2027.02)**: PRS-12, INB-04, PRC-05, SHP-08, AI 관련 (별도)
**Phase 4 (Integration, 2027.03-04)**: INB-08, PRC-06~07, USR-06

### 4.2 Must Have 핵심 기능 총 개수
- INB: 6개
- PRS: 12개 (예지보전 AI 포함)
- PRC: 4개
- SHP: 7개
- MST: 9개
- USR: 5개 + 2개(M)
- **합계: 약 45개 Must Have 기능**

## 5. 의존성 그래프

```
[MST] 기준정보 ─┬─→ [INB] 입고관리 ─┬─→ [PRS] 프레스 ─→ [PRC] 후공정 ─→ [SHP] 출하
                │                    │                                      │
                └─→ [USR] 사용자관리 ─┴─────────────────────────────────────┘
                                                              ↓
                                                  [AI Agent / KPI / Dashboard]
```

- **기준정보(MST)는 모든 트랜잭션의 선행조건** → Phase 1에서 100% 완성 필수
- 입고(INB)와 출하(SHP)는 LOT 추적의 양 끝점 → 데이터 연결 무결성 핵심

## 6. 비기능 요구사항 (NFR)

| 영역 | 요구사항 |
|------|----------|
| 성능 | 화면 전환 < 2초, IoT 데이터 입력 지연 < 1초 |
