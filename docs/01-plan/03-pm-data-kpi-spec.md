```
ERP (더존)         ─REST───→                                          ─→ ML Training
                                                    Object Storage         (MLflow)
                                                    (Audit Log, Files)
                                                                      ─→ AI Agent (RAG)
                                                                          (Vector DB)
```

## 4. KPI 체계 (KPI Framework)

### 4.1 KPI 계층 구조

```
[Level 1: 경영 KPI]  ← 대표/공장장
   ├─ OEE (설비종합효율)
   ├─ 납기 준수율 (OTD)
   ├─ 불량률 (PPM)
   └─ 1인당 생산성

[Level 2: 부서 KPI]  ← 부서장
   ├─ 생산: 가동률, 사이클 타임
   ├─ 품질: 공정별 불량률, Cpk
   ├─ 보전: MTBF, MTTR, 예지보전 적중률
   └─ 자재: 재고 회전율, 공급처 품질

[Level 3: 현장 KPI]  ← 작업반장/작업자
   ├─ 시간당 생산량
   ├─ 자주검사 합격률
   ├─ 4M 변경 준수
   └─ 정지 시간 분류별
```

### 4.2 핵심 KPI 정의

#### 4.2.1 생산성 KPI

| KPI | 정의 | 계산식 | 목표 |
|-----|------|--------|------|
| **OEE** | 설비종합효율 | 가동률 × 성능률 × 양품률 | 82% |
| 가동률 (Availability) | 계획 대비 실제 가동시간 | 실가동시간 / 계획가동시간 | 90% |
| 성능률 (Performance) | 표준 사이클 대비 실제 성능 | (이론 SPM × 실가동시간) / 실제 생산수량 | 95% |
| 양품률 (Quality) | 총생산 중 양품 비율 | 양품수량 / 총생산수량 | 99.2% |
| 시간당 생산량 | 시간당 양품 생산 수 | 양품수량 / 시간 | 부품별 표준 |
| 사이클 타임 | 1개 부품 생산 시간 | 60 / SPM | 부품 표준 |

#### 4.2.2 품질 KPI

| KPI | 정의 | 계산식 | 목표 |
|-----|------|--------|------|
| **PPM** | 백만개당 불량 | 불량수량 / 총생산수량 × 1,000,000 | 800 |
| 공정 능력 (Cp) | 공정 산포 | (USL - LSL) / 6σ | ≥ 1.33 |
| 공정 능력 (Cpk) | 편향 포함 능력 | min((USL-μ)/3σ, (μ-LSL)/3σ) | ≥ 1.33 |
| 첫 통과 수율 (FTY) | 한 번에 합격한 비율 | 양품 / (양품 + 재작업) | 98% |
| 공급처 불량률 | 입고 검사 불합격률 | 불합격 LOT / 총 입고 LOT | 0.5% |
| 클레임 건수 | 모회사 클레임 월별 | - | 0 |

#### 4.2.3 보전 KPI (예지보전 효과)

| KPI | 정의 | 계산식 | 목표 |
|-----|------|--------|------|
| MTBF | 평균 고장 간격 | 총 가동시간 / 고장횟수 | 720시간 |
| MTTR | 평균 수리 시간 | 총 수리시간 / 수리횟수 | < 1시간 |
| 예지보전 적중률 | AI 예측이 실제 고장으로 이어진 비율 | True Positive / 전체 예측 | ≥ 70% |
| 돌발 다운타임 | 계획되지 않은 정지 시간 | - | 월 3시간 이하 |
| 정비 비용 절감 | 예지보전으로 절감된 비용 | - | 정성 측정 |

#### 4.2.4 출하/물류 KPI

| KPI | 정의 | 계산식 | 목표 |
|-----|------|--------|------|
| 납기 준수율 (OTD) | 납기 내 출하 비율 | 납기준수건 / 총 출하건 | 99% |
| LOT 추적 응답시간 | 추적 요청 → 보고서 생성 | - | < 30초 |
| 출하 처리 속도 | 박스 1개당 출하 처리 시간 | - | < 3초 |

## 5. 대시보드 화면 기획

> 사업계획서 p41 "AI 대시보드" 모듈의 구체화. 사용자 역할별 4개 대시보드 + 통합 KPI 대시보드.

### 5.1 대시보드 종류

| 대시보드 | 대상 사용자 | 갱신 주기 | 핵심 위젯 |
|----------|------------|----------|----------|
| **D1: 경영 대시보드** | 대표/공장장 | 5분 | OEE, 납기, PPM, 매출 추이 |
| **D2: 생산 현황 대시보드** | 생산팀장/작업반장 | 1초 | 라인별 현황, 진행 WO, 정지 알람 |
| **D3: 설비 상태 모니터링** | 보전팀 | 1초 | 8대 프레스 진동/온도/전류 라이브 |
| **D4: 품질 현황 대시보드** | 품질팀 | 1분 | SPC 차트, 불량 Top-N, 4M 변경 |
| **D5: 출하 현황 대시보드** | 출하/영업 | 5분 | 납기 임박, 출하 진행, 클레임 추적 |

### 5.3 D2: 생산 현황 대시보드 (Production Monitor)

**핵심 위젯**

**핵심 위젯**

- 예지보전 AI 예측 결과 (각 설비별 향후 7일 고장 확률)
- 정비 이력 타임라인
- 금형 수명 게이지 (잔여 타수)

### 5.5 D4: 품질 현황 대시보드 (Quality Monitor)

**핵심 위젯**
- 공정별 PPM 히트맵

- 불량 Top-10 (유형별/공정별)
- Cpk 분포
- 4M 변경 vs 불량률 오버레이 차트
- 모회사 클레임 트래커

### 5.6 D5: 출하 현황 대시보드 (Shipment Monitor)

**핵심 위젯**
- 오늘 출하 계획 vs 실적
- 납기 임박 24시간 내 건 (빨강/노랑)
- 차량별 출하 진행 상황
- 월별 OTD 추이
- 클레임 발생 LOT 추적

## 6. 데이터 파이프라인 상세

### 6.1 실시간 파이프라인 (Real-time)
```
PLC/Sensor → Edge Gateway (OPC-UA → MQTT) → MQTT Broker (EMQX)
   → Stream Processor (Telegraf 또는 사내 Python Worker)
   → TimescaleDB (Hypertable, 1초 단위)
   → WebSocket Broadcaster → Frontend Live Chart
```

### 6.2 배치 파이프라인 (Batch)
```
TimescaleDB → Hourly Aggregation Job → PostgreSQL (OLTP)
   → Daily ETL → S3 Parquet (Data Lake)
   → Weekly Aggregation → BI Layer
```

### 6.3 ML 학습 파이프라인
```
S3 Parquet (학습 데이터) → MLflow / DVC
   → Training Job (Python, scikit-learn / PyTorch)
   → Model Registry → Serving (FastAPI)
   → Predictions → PostgreSQL → Dashboard
```

## 7. 데이터 거버넌스 정책

### 7.1 데이터 품질 (DQ)
- **완전성**: 모든 IoT 채널 결측률 < 1% (SLA)
- **정확성**: 자주검사 입력값 → 자동 검증 룰 (단위, 범위)
- **일관성**: 코드 마스터 단일화 (불량코드 등)
- **적시성**: 실시간 데이터 지연 < 5초

### 7.2 데이터 보안
- 전송 구간: TLS 1.3
- 저장: 컬럼 단위 암호화 (PII)
- 접근 제어: RBAC + Row-level Security
- 감사 로그: 모든 조회/변경 기록 (불변)

### 7.3 데이터 백업/복구
- PostgreSQL: 1일 1회 풀백업 + WAL 연속 백업
- TimescaleDB: 1일 1회 + 압축
- S3: 버전 관리 + Cross-region 복제
- RTO 4시간, RPO 1시간

### 7.4 데이터 보존 및 폐기
- 개인정보(작업자 사번): 퇴사 후 5년 보관 후 마스킹
- IoT 시계열: 위 보존 정책 (3장 참조)
- 감사 로그: 영구
