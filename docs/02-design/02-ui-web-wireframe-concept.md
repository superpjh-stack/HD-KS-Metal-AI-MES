# 웹 UI 와이어프레임 개념 설계 (Web UI Wireframe Concept)

> 작성자: UI-Web (디자이너 2) | 작성일: 2026-05-11 | 문서 버전: v1.0
> 분류: PDCA - Design Phase - Web UI
> 대상 디바이스: Desktop (1920×1080 이상), 사무실 환경

---

## 1. 문서 목적

본 문서는 광성정밀 AI-MES 웹 UI의 **레이아웃, 와이어프레임 개념, 핵심 컴포넌트, 화면별 표준 패턴**을 정의한다.
ASCII 와이어프레임 형식으로 구조를 표현하며, 향후 Figma 기반 고해상도 디자인 작업의 기준이 된다.
정보 아키텍처는 `01-ux-information-architecture.md`를 참조한다.

## 2. 디자인 시스템 (Design System) 개요

### 2.1 그리드 시스템
- 12 컬럼 그리드, 컬럼 폭 92px, 거터 24px (1440px 기준)
- 반응형 브레이크포인트:
  - sm: 768px (사무실 노트북 - 최소 지원)
  - md: 1280px (표준 PC)
  - lg: 1440px (메인 타깃)
  - xl: 1920px (Full HD 모니터)
  - 2xl: 2560px (대형 디스플레이 - D1 경영 대시보드)

### 2.2 표준 컴포넌트 라이브러리 (Component Library)

| 컴포넌트 | 용도 | 변형 |
|---------|------|------|
| Button | 액션 | Primary / Secondary / Danger / Ghost / Icon-only |
| Card | 정보 그룹 | Default / KPI / Status / Compact |
| Table | 데이터 목록 | Standard / Striped / Sticky-header / Editable |
| Modal | 상세 입력 | Small / Medium / Large / Fullscreen |
| Toast | 알림 | Success / Warning / Error / Info |
| Badge | 상태 표시 | Pill / Dot / Count |
| Chart | 시각화 | Line / Bar / Pie / Gauge / Heatmap / SPC |
| FormField | 입력 | Text / Number / Select / Date / File / QR / Voice |
| Tabs | 화면 전환 | Underline / Pill |
| Drawer | 보조 패널 | Right / Bottom |
| Tree | 계층 데이터 | Master Data / Sitemap |
| Avatar | 사용자 | User / Equipment / Part |

## 4. 인터랙션 패턴 (Interaction Patterns)

### 4.1 표준 인터랙션

| 액션 | 트리거 | 피드백 |
|------|--------|--------|
| 데이터 저장 | 저장 버튼 | Toast 2초 + 행 하이라이트 |
| 데이터 삭제 | 삭제 버튼 | Confirm 모달 → Toast + 5초 Undo |
| 행 클릭 | 테이블 행 | 상세 Drawer 또는 새 페이지 |
| QR 스캔 | 카메라 활성 | 비프음 + 화면 점멸 + 자동 진행 |
| 알람 발생 | WebSocket Push | Toast(우상단) + 소리 + 깜빡임 |
| 차트 호버 | 마우스 오버 | 툴팁 (시각, 값, 추가 정보) |
| 글로벌 검색 | Cmd/Ctrl+K | 명령 팔레트 모달 |

### 4.2 키보드 단축키 (전문가 사용자용)

| 단축키 | 동작 |
|--------|------|
| Cmd/Ctrl + K | 글로벌 검색 |
| Cmd/Ctrl + / | AI Agent 토글 |
| Cmd/Ctrl + S | 저장 |
| Cmd/Ctrl + N | 새로 만들기 |
| Esc | 모달/Drawer 닫기 |
| F | 전체화면 (대시보드) |

## 5. 반응형 전략 (Responsive Strategy)

### 5.1 브레이크포인트별 동작

| 브레이크포인트 | 사이드바 | 콘텐츠 |
|---------------|---------|--------|
| 2xl (2560+) | 펼쳐짐 (240px) | 위젯 4×N 그리드 |
| xl (1920) | 펼쳐짐 (240px) | 위젯 4×N 그리드 |
| lg (1440) | 펼쳐짐 (220px) | 위젯 3×N 그리드 |
| md (1280) | 접힘 (64px, 아이콘만) | 위젯 2~3×N |
| sm (768) | 모바일 메뉴 (햄버거) | 위젯 1~2×N |

### 5.2 차트 반응성
- 모든 차트는 SVG 기반, 부모 컨테이너 너비 100% 추종
- 축 라벨 회전 (좁을 때 45°)
- 데이터포인트 > 200개일 때 다운샘플링

## 6. 컴포넌트 명세 - 핵심 위젯 상세

### 6.1 KPICard 컴포넌트
- Props: `title`, `value`, `unit`, `trend` (`up`/`down`/`flat`), `delta`, `colorTone`
- 사이즈: 220px × 140px (xl 기준)
- 큰 숫자(48px Bold) + 상단 라벨(14px) + 하단 델타(14px)

### 6.2 StatusCard (설비 카드)
- Props: `equipmentId`, `status` (4단계), `metrics`, `alerts`
- 클릭 시 D3 상세 화면 딥링크
- 5초마다 WebSocket 업데이트

### 6.3 LiveChart
- Library: ECharts (성능 우선)
- 윈도우: 60초 / 5분 / 1시간 토글
- 더블클릭 시 풀스크린 분석 화면

### 6.4 AlertToast
- 위치: 우상단 스택 (최대 5개)
- 자동 제거: Info 5초 / Warning 10초 / Error/Critical 수동
- 클릭 시 해당 페이지 이동

## 7. 빈 상태(Empty State) 및 에러(Error) 디자인

- **빈 상태**: 일러스트 + 안내 문구 + 액션 버튼 ("아직 데이터가 없습니다. 첫 LOT 등록하기 →")
- **에러 상태**: 친절한 한국어 메시지 + 재시도 버튼 + AI Agent로 문의 링크
- **로딩**: Skeleton UI (회색 깜빡임), 데이터 도착 시 페이드인
- **권한 없음**: "이 화면을 볼 권한이 없습니다. 관리자에게 문의하세요."
