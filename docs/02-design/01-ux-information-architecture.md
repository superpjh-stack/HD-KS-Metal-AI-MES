## 2. UX 핵심 원칙 (UX Principles)

### 2.1 5대 원칙

| # | 원칙 | 설명 | 구현 예 |
|---|------|------|---------|
| 1 | **Floor-First (현장 우선)** | 사무실 UI보다 현장 작업자 UI가 더 중요 | 큰 글씨, 장갑 끼고도 누르는 버튼 (최소 64dp) |
| 2 | **One-Action Per Screen (한 화면 한 행동)** | 현장 작업자는 한 번에 한 가지만 | 입고 등록, 자주검사 등 단일 목적 화면 |
| 3 | **Glanceable (한눈에)** | 3초 안에 상태 파악 가능 | 색상 + 아이콘 + 큰 숫자, 텍스트 최소화 |
| 4 | **Forgiving (실수 허용)** | 잘못 눌러도 회복 가능 | 5초 Undo, 명확한 취소, 확인 다이얼로그 |
| 5 | **Context-Aware (맥락 인식)** | 사용자 위치/시간/역할 자동 인식 | 사번 로그인 시 해당 라인 자동 표출 |

### 2.2 디자인 토큰 가이드

| 토큰 | 값 | 사용처 |
|------|------|--------|
| Primary Color | #1E40AF (Indigo 800) | 주요 액션 |
| Success | #16A34A (Green 600) | 정상, 합격 |
| Warning | #F59E0B (Amber 500) | 주의, 한계 임박 |
| Danger | #DC2626 (Red 600) | 이상, 불합격 |
| Neutral | #64748B (Slate 500) | 보조 정보 |
| Font (Web) | Pretendard / Noto Sans KR | 본문 14-16px |
| Font (Floor Tablet) | Pretendard Bold | 본문 18-24px (큰 글씨) |
| Touch Target Min | 64×64dp | 현장 모바일 |
| Border Radius | 8px (Card), 4px (Button) | - |

### 2.3 접근성 (Accessibility)
- WCAG 2.1 AA 준수
- 색상 + 아이콘 동시 사용 (색맹 고려)
- 다크 모드 (야간 교대조용)
- 키보드 네비게이션 (PC)
- 음성 입력 (현장 소음 환경 고려)

## 3. 사용자 페르소나 (Personas)

### 3.1 페르소나 1: 김작업 (현장 프레스 오퍼레이터)
| 속성 | 내용 |
|------|------|
| 나이/경력 | 52세, 25년 경력 |
| 디지털 친숙도 | 낮음 (카톡, 유튜브 정도) |
| 사용 시간 | 교대 시작 시 5분 + 시간당 1분 |
| 사용 디바이스 | 프레스 옆 거치형 태블릿 10인치 |
| 주요 작업 | 작업 시작/종료, 자주검사, 불량 등록, 정지 사유 |
| 환경 제약 | 소음, 기름때, 장갑, 시야 제한 |
| Goal | "빨리 입력하고 작업에 집중하고 싶다" |
| Pain | "버튼이 너무 작다, 글씨가 안 보인다, 절차가 복잡하다" |

### 3.2 페르소나 2: 박반장 (작업반장, 50대 후반)
| 속성 | 내용 |
|------|------|
| 디지털 친숙도 | 중간 (엑셀 가능) |
| 사용 시간 | 교대 시작 30분 + 수시 |
| 사용 디바이스 | 사무실 PC 24인치 + 휴대폰 |
| 주요 작업 | 라인 모니터링, 인원 배치, 정지 대응, 교대 보고 |
| Goal | "8대 프레스가 다 잘 돌아가는지 한눈에 보고, 문제 생기면 즉시 알고 싶다" |
| Pain | "여기저기 흩어진 정보를 모으는 데 시간 낭비" |

### 3.3 페르소나 3: 이품질 (품질 엔지니어, 30대)
| 속성 | 내용 |
|------|------|
| 디지털 친숙도 | 높음 (Excel 매크로, Minitab) |
| 사용 디바이스 | 사무실 PC + 검사실 노트북 |
| 주요 작업 | SPC 분석, 불량 원인 추적, 모회사 응대, 4M 관리 |
| Goal | "데이터로 원인을 빨리 찾고 싶다, 클레임 대응 자료를 즉시 만들고 싶다" |
| Pain | "ERP/MES/검사기 데이터를 일일이 다운받아 엑셀로 합치는 게 너무 오래 걸린다" |

### 3.4 페르소나 4: 최보전 (보전 담당, 40대)
| 속성 | 내용 |
|------|------|
| 디지털 친숙도 | 중상 (PLC, 전기도면) |
| 사용 디바이스 | 사무실 PC + 현장 휴대폰 |
| 주요 작업 | 정비 계획, 돌발고장 대응, 부품 발주 |
| Goal | "고장 나기 전에 미리 알고 싶다, 정비 효과를 데이터로 보여주고 싶다" |

### 3.5 페르소나 5: 정공장 (공장장/CTO, 50대)
| 속성 | 내용 |
|------|------|
| 사용 디바이스 | 사무실 27인치 + 휴대폰 |
| 주요 작업 | 일일 보고 확인, 의사결정, 모회사 대응 |

### 3.6 페르소나 6: 한품질 (모회사 SQA, 외부 감사인)

## 5. 내비게이션 패턴 (Navigation Pattern)

### 5.1 웹 (Desktop)

| 영역 | 구성 |
|------|------|
| Top Bar | 로고 / 회사명, 글로벌 검색, AI Agent 버튼, 알림 (🔔), 사용자 |
| Side Nav (Left) | 9개 대분류 메뉴, 펼치기/접기, 즐겨찾기 |
| Breadcrumb | 홈 > 프레스 공정관리 > 실시간 모니터링 |
| Action Bar (상단 우측) | 새로고침, 기간 선택, 내보내기 |
| Status Bar (하단) | 연결 상태, 마지막 갱신 시각 |

### 5.2 태블릿 (Floor)
- 하단 탭 바 (최대 5개)
- 상단 컨텍스트: 작업자 이름, 라인 번호, 시각
- 큰 카드 그리드 (한 화면 4~6개 카드)
- AI Agent: 우하단 음성 버튼 (마이크)

### 5.3 휴대폰 (Mobile - 반장/관리자)
- 하단 탭 4개: 홈 / 알림 / AI / 더보기
- 푸시 알림 클릭 → 해당 화면 딥링크

## 6. 사용자 여정 맵 (User Journey Maps)

### 6.1 Journey 1: 김작업의 하루 (프레스 오퍼레이터)

```
[06:00 출근]
   ↓
[06:10] 태블릿 사번 로그인
   → "안녕하세요 김작업님, 오늘은 PRESS-03 담당입니다"
   ↓
[06:12] 오늘의 작업 지시 확인
   → WO-20260511-A, 부품 P-1234, 목표 5,000개
   → 표준 조건 (타발력 120톤, SPM 60)이 큰 카드로 표시
   ↓
[06:15] 금형 확인 → "MOLD-A057 장착 OK" 체크
   ↓
[06:20] 작업 시작 버튼
   → 큰 초록 버튼, 카운터 시작
   ↓
[07:00] 1시간 자주검사 알림 (진동/소리)
   → 폭/두께 측정 후 입력
   → "합격" 표시, 다음 알림 1시간 후
   ↓
[09:30] AI 알람 표출
   → "PRESS-03 진동이 평소보다 1.5배, 금형 점검 권고"
   → "확인", "정비 호출" 2개 큰 버튼
   ↓
[09:40] 점검 결과 정상 → "정상" 입력
   ↓
[12:00] 식사 정지 → 정지 사유 "식사" 선택
   ↓
[15:00] 불량 발생 → "불량 등록"
   → 카메라로 사진 1장, 수량 3, 유형 "찍힘"
   ↓
[15:02] AI Agent 음성: "지난 30분간 진동 데이터 보면 평소와 다른지?"
   → AI 답변: "정상 범위입니다. 자재 LOT 확인을 권고합니다."
   ↓
[18:00] 작업 종료
   → 자동 인수인계 메모 생성 → 교대자 화면에 자동 표출
```

**Pain Points & Solutions**
| 시점 | Pain | Solution |
|------|------|----------|
| 자주검사 입력 | 키보드 입력 어려움 | 음성 입력 / 큰 숫자 키패드 |
| 불량 사진 첨부 | 손이 더러움 | 음성 명령으로 카메라 호출 |
| AI 알람 | 너무 잦으면 무시 | 알람 우선순위 시스템, 통계 |

### 6.3 Journey 3: 이품질의 클레임 대응

```
[09:00] 모회사 클레임 메일 수신

[09:02] AI-MES 글로벌 검색에 LOT 입력

   - 4M 변경 이력 (해당일 변경 없음)
   - 외주 도금업체 (KS도금, LOT-G-001)
   - 출하 박스 번호 / 차량번호 / 납품 시각
   ↓
[09:10] 의심 가는 외주 도금 LOT 확인
   → AI Agent: "동일 외주 LOT으로 출하된 다른 박스가 있는지?"
   → AI: "동일 LOT 미회수 30박스 식별, 모회사 통보 권고"
   ↓
[10:00] 보고서 PDF 모회사 송부 + 회수 조치
```

### 6.4 Journey 4: 모회사 SQA 감사 시나리오

```
[감사 당일 14:00] 회의실 입장
   ↓
[14:05] 모회사 SQA: "임의의 LOT 추적 시연 부탁드립니다"
   → 임의 LOT 번호 제시

   ↓
[14:15] SQA: "감사 통과"
```

## 7. 화면 간 흐름 다이어그램 (Screen Flow)

### 7.1 입고 시나리오
```
[로그인]
  → [입고 등록(QR)]
  → [검사 결과 입력]
  → [합격: LOT 발번 / 불합격: 격리 등록]
  → [입고 완료 알림 → 생산 계획 자동 갱신]
```

### 7.2 프레스 작업 시나리오
```
[작업자 로그인]
  → [작업지시 확인]
  → [표준 조건 확인]
  → [작업 시작]
  → [(시간당) 자주검사 입력]
  → [(이상 시) 정지/불량 등록]
  → [작업 종료]
  → [교대 인수인계 자동 생성]
```

## 8. 핵심 UX 결정사항 (UX Decisions)

| # | 결정 | 사유 |
|---|------|------|
| UX-D1 | 현장 태블릿은 "한 화면 한 행동" 원칙 강제 | 작업자 인지 부담 최소화 |
| UX-D2 | AI Agent는 모든 화면에서 우하단 Floating Button | 일관된 접근, 자연어 도움 |
| UX-D3 | 알람은 4단계 (Info/Warning/Error/Critical) + 색상 + 사운드 | 노이즈 방지 |
| UX-D4 | 다크 모드 야간 교대조에 자동 활성화 (22:00~06:00) | 눈 피로 |
| UX-D5 | 모든 액션은 5초 Undo 제공 (실수 회복) | Forgiving 원칙 |
| UX-D6 | 한국어 우선, 베트남어 토글 가능 (외국인 작업자) | - |
| UX-D7 | 음성 입력 = 보조, 항상 텍스트 확인 단계 거침 | 오인식 방지 |
| UX-D8 | "오프라인 모드" 명확히 표시, 자동 동기화 | 네트워크 단절 대비 |
