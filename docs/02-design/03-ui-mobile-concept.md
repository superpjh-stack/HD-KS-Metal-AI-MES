# 공장 현장 모바일/태블릿 UI 개념 설계 (Floor Mobile UI Concept)

> 작성자: UI-Mobile (디자이너 3) | 작성일: 2026-05-11 | 문서 버전: v1.0
> 분류: PDCA - Design Phase - Mobile/Tablet UI
> 대상 디바이스: 산업용 태블릿 10" (Zebra ET40 등), 스마트폰 (반장/관리자)

---

## 1. 문서 목적

본 문서는 광성정밀 AI-MES의 **현장(Floor) 사용자용 모바일/태블릿 UI** 개념을 정의한다.
사무실 PC 환경과 근본적으로 다른 제약(소음, 장갑, 기름때, 시야 등)을 고려한 별도의 디자인 원칙을 적용한다.
사무실 웹 UI는 `02-ui-web-wireframe-concept.md`를 참조한다.

## 3. 현장 UX 핵심 원칙 (Floor UX Principles)

### 3.1 "Hand-eye-noise" 트리플 제약

| 제약 | 의미 | 디자인 대응 |
|------|------|------------|
| Hand (장갑·기름) | 정밀 터치 불가 | 최소 터치 영역 64×64 dp, 버튼 간 16dp 마진 |
| Eye (시야 제한) | 안전모, 보안경, 거리 1.5m | 24sp+ 폰트, 고대비 (다크모드 권장) |
| Noise (소음) | 80~95 dB 프레스 소음 | 진동·시각 알람 우선, 사운드는 보조 |

### 3.2 추가 원칙

| # | 원칙 | 설명 |
|---|------|------|
| 1 | **Big & Bold** | 모든 UI 요소를 PC의 2배 크기로 |
| 2 | **One-Hand Friendly** | 한 손으로 조작 가능한 버튼 배치 (휴대폰) |
| 3 | **Glove Mode** | 정전식 장갑 가정, 압력 임계 낮춤 |
| 4 | **Always Online + Offline-First** | 네트워크 단절 시에도 입력 가능, 자동 동기화 |
| 5 | **Voice-Friendly** | 손이 더러우면 음성 입력 가능 |
| 6 | **Status-First** | 가장 큰 시각 요소는 항상 "현재 상태" |
| 7 | **Big Numbers** | 숫자는 굵게 + 크게, 라벨은 보조 |

## 7. 오프라인 모드 (Offline-First)

### 7.1 동작 원리
- 모든 입력은 IndexedDB(PWA) 또는 SQLite(Native)에 우선 저장
- 네트워크 복구 시 자동 동기화 (FIFO 큐)
- 동기화 충돌 시 서버 우선 + 사용자 알림

### 7.2 UI 표시

```
┌────────────────────────────┐
│ 📡 오프라인 모드 (12건 대기)│ ← 상단 노란 바
├────────────────────────────┤
│ (정상 화면)                │
└────────────────────────────┘
```

- 오프라인 시 일부 기능 비활성화 (예: LOT 추적 조회)
- 입력만 허용, "동기화 대기 중" 표시

## 8. 음성 인터페이스 (Voice UI)

### 8.1 음성 입력 시나리오

| 시나리오 | 발화 예시 | 인식 → 변환 |
|---------|-----------|------------|

- 알람: "주의, 진동 이상 발생"
- 확인 응답: "30.1 입력되었습니다"
- 시끄러운 환경 대비 200% 볼륨 옵션

### 8.3 기술 스택 (안)
- 인식: Google Cloud STT (한국어, 베트남어) 또는 OpenAI Whisper
- TTS: Google Cloud TTS 또는 ElevenLabs
- 오프라인 대비 사내 Whisper.cpp 옵션 검토

## 9. 인증 및 사용자 컨텍스트

### 9.1 로그인 방식

| 방식 | 사용자 | 디바이스 |
|------|--------|---------|
| RFID 카드 + PIN | 프레스 오퍼레이터 | 거치 태블릿 |
| 사번 + 패스워드 | 검사원, 출하담당 | 휴대 태블릿 |
| 사번 + 푸시 인증 | 반장, 관리자 | 휴대폰 |
| SSO (사내 AD) | 사무직 | PC |

### 9.2 컨텍스트 자동 인식
- 거치 태블릿: 디바이스 → 라인 → 작업자 (RFID 인증)
- 휴대 디바이스: 작업자 → 권한별 화면
